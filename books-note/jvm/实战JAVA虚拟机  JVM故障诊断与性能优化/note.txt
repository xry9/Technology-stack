3.1.2 类加载／卸载的跟踪
  随着动态代理、 AOP等技术的普遍使用，系统也极有可能在运行时动态生成某些类，这些类相对比较隐敲，无法通过文件系统找到 ，此时虚拟机提供的类加载／卸载跟踪参数
就显得格外有意义

-XX:+TraceClassUnloading -XX:+TraceClassLoading
Java 虚拟机还允许研发人员在运行时打印、 查看系统中类的分布情况，只要在系统启动时加上 XX int assHistogr 参数，然后在 Java 的控制台中按下 Ctr Break 组合键，控制台上就会显示当前的类信息柱状图
